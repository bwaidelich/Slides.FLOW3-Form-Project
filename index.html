What are forms?

Wikipedia: "A webform on a web page allows a user to enter data that is sent to a server for processing."

Forms are bad

  • Dumb (no proper validation)
  • Oblivious (forget their state & values after submission)
  • Wasteful (upload 1 Gigabyte and realize that this exceeds the allowed file size afterwards)
  • Stingy (only a couple of native form elements)
  • Stubborn (try styling form elements to look the same on every browser)

 

* XForms promises remedy for some of the issues

Forms are important

Interactivity gets more and more important
Forms are still the main way to enable client to server data transmission

Many use cases:

  • Contact/mail forms
  • Registration/Login
  • Survey
  • Setup TYPO3 Phoenix Install-Tool
  • Create/Edit records TCEForms reloaded

Anatomy of a FLOW3 Form

Glossary

FormDefinition
Domain model of the Form
Page
Step of a form, each form consists of at least one page
FormElement
The actual input element. FormElements are attached to pages or sections
Section
A container element that groups a list of FormElements, e.g. fieldset
Finisher
PHP Code that is execute after the last page of a form has been submitted
ProcessingRule
Property mapping & validation rules for one or more FormElements
FormRuntime
The current state of a form with all its submitted data
Renderer
Renders the FormDefinition and all it's elements
FormFactory
Builds a FormDefinition

Classes/TestFormFactory.php

								class TestFormFactory extends \TYPO3\Form\Factory\AbstractFormFactory {

								  /**
								  * @param array $configuration
								  * @param string $presetName
								  * @return \TYPO3\Form\Core\Model\FormDefinition
								  */
								  public function build(array $configuration, $presetName) {
								    $formConfiguration = $this->getPresetConfiguration($presetName);
								    $formDefinition = new \TYPO3\Form\Core\Model\FormDefinition('contactForm', $formConfiguration);

								    $page1 = $formDefinition->createPage('page1');

								    $name = $page1->createElement('name', 'TYPO3.Form:SingleLineText');
								    $name->setLabel('Name');
								    $name->addValidator(new \TYPO3\FLOW3\Validation\Validator\NotEmptyValidator());

								    $email = $page1->createElement('email', 'TYPO3.Form:SingleLineText');
								    $email->setLabel('Email');
								    $email->addValidator(new \TYPO3\FLOW3\Validation\Validator\NotEmptyValidator());
								    $email->addValidator(new \TYPO3\FLOW3\Validation\Validator\EmailAddressValidator());

								    $comment = $page1->createElement('comment', 'TYPO3.Form:MultiLineText');
								    $comment->setLabel('Comment');
								    $comment->addValidator(new \TYPO3\FLOW3\Validation\Validator\NotEmptyValidator());
								    $comment->addValidator(
								      new \TYPO3\FLOW3\Validation\Validator\StringLengthValidator(array('minimum' => 3))
								    );

								    return $formDefinition;
								  }
								}

FormElement

The FormElement is represented by the actual input element but it can be any HTML/JavaScript snippet.

Provided FormElements (version 1.0):

Text input
SingleLineText, MultiLineText
Selection
Checkbox, SingleSelectDropdown, MultipleSelectDropdown, MultipleSelectCheckboxes
Widgets
DatePicker, FileUpload, ImageUpload
Container elements
Section, Page, PreviewPage

Custom FormElements

Examples of custom FormElements:

  • Mass uploader
  • Date/Time picker
  • Conditional Elements
  • Scripts (e.g. collect browser details)

Finisher

A Finisher is a piece of code that is executed when the last page of the form has been submitted successfully.

Examples of Finishers:

  • Email finisher
  • Write form values to Database
  • Redirect to some action
  • Execute arbitrary code (Closure)

Renderer

A Form Renderer outputs a FormDefinition and/or single FormElements to the client.

Examples of custom Renderers:

  • Flash/Silverlight based forms
  • Forms for mobile clients
  • Integrate other Templating Engines
  • CLI based forms

	var supports3DTransforms =  document.body.style['webkitPerspective'] !== undefined ||
					document.body.style['MozPerspective'] !== undefined ||
					document.body.style['perspective'] !== undefined;

	function linkify( selector ) {
	    if( supports3DTransforms ) {

	        var nodes = document.querySelectorAll( selector );

	        for( var i = 0, len = nodes.length; i < len; i++ ) {
	            var node = nodes[i];

	            if( !node.className || !node.className.match( /roll/g ) ) {
	                node.className += ' roll';
	                node.innerHTML = '' + node.innerHTML + '';
	            }
	        };
	    }
	}

	linkify( 'a' );
